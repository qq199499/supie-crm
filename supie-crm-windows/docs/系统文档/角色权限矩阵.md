# 八角色权限矩阵建议稿

## 1. 文档说明

- 文档日期：2026-04-03
- 适用系统：项目过程管理系统 / CRM
- 文档目标：基于最新角色设置，整理一版可评审、可落地的权限矩阵草案
- 角色范围：`admin`、`management`、`sales`、`pm`、`implementer`、`finance`、`sales_director`、`project_director`

本文档基于当前业务诉求整理，适用于产品、研发、测试一起确认权限边界。

当前建议仍按三层设计：

1. 菜单权限：决定模块是否可见
2. 操作权限：决定是否可新增、编辑、删除、导出、审批
3. 数据权限：决定可见数据范围

## 2. 角色定义


| 角色   | 代码                 | 角色定位                                       |
| ---- | ------------------ | ------------------------------------------ |
| 管理员  | `admin`            | 系统维护、权限配置、全局业务兜底，拥有最高权限                    |
| 管理层  | `management`       | 查看全局经营状态，只读查看，不进行业务处理                      |
| 销售   | `sales`            | 维护本人负责的客户、商机、合同，并推进回款                      |
| 项目经理 | `pm`               | 仅查看自己负责或参与的项目；仅对自己是项目经理的项目进行管理，并跟进项目交付过程   |
| 实施人员 | `implementer`      | 参与本人项目的执行，仅可以填写负责人为自己的任务、里程碑、风险等内容，且没有新建权限 |
| 财务   | `finance`          | 查看全局数据，仅管理开票                               |
| 销售总监 | `sales_director`   | 查看并管理销售线全部核心业务，并负责赢单、签约审批                  |
| 项目总监 | `project_director` | 查看并管理全部项目，并负责项目结项审批                        |


## 3. 权限级别说明

### 3.1 菜单权限标记


| 标记   | 含义              |
| ---- | --------------- |
| `管理` | 菜单可见，且可进行主要业务操作 |
| `查看` | 菜单可见，以浏览和查询为主   |
| `处理` | 菜单可见，且允许执行审批等动作 |
| `无`  | 菜单不可见           |


### 3.2 数据权限标记


| 标记     | 含义                      |
| ------ | ----------------------- |
| `全部`   | 全公司全部数据                 |
| `本人负责` | 当前用户作为负责人、owner 或主责人的数据 |
| `本人参与` | 当前用户参与的项目或与其直接关联的数据     |
| `本人相关` | 当前用户发起、跟进、提交或关联的记录      |
| `无`    | 无数据权限                   |

## 7. 角色边界说明


| 角色                 | 建议边界                                                    |
| ------------------ | ------------------------------------------------------- |
| `admin`            | 保留系统最高权限，可查看并处理所有事务和内容                                  |
| `management`       | 只看不做，查看系统所有数据，但不进行任何编辑、提交或审批操作                          |
| `sales`            | 只处理负责人为自己的客户、商机、合同及回款应收，并可查看关联项目                        |
| `pm`               | 只查看自己负责或参与的项目；仅处理项目经理为自己的项目，并查看与该项目关联的客户、商机、合同及回款应收     |
| `implementer`      | 只看项目成员为自己的项目，且仅可编辑负责人为自己的任务、里程碑、风险，不允许新建                |
| `finance`          | 查看系统所有数据，但仅管理开票                                         |
| `sales_director`   | 查看并管理全部客户、商机、合同、回款应收；可查看全部项目和开票，不能管理项目和开票；负责商机赢单与合同签约审批 |
| `project_director` | 查看并管理全部项目；可查看全部客户、商机、合同、回款应收及开票，不能管理这些模块；负责项目结项审批       |


## 8. 审批规则建议


| 审批事项 | 审批角色 | 说明                   |
| ---- | ---- | -------------------- |
| 商机赢单 | 销售总监 | 销售提交后，由销售总监审批通过或驳回   |
| 合同签约 | 销售总监 | 合同提交后，由销售总监审批通过或驳回   |
| 项目结项 | 项目总监 | 项目经理提交后，由项目总监审批通过或驳回 |

## 11. 建议的下一步

1. 先确认 8 个角色的菜单权限是否成立
2. 再确认每个角色的数据范围是否按“本人负责 / 本人参与 / 全部”拆分
3. 最后再决定是否补充部门、区域、分公司等更复杂的数据权限层级
