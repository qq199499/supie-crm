# 项目文档助手

## 适用范围

用于当前仓库的文档补全、重写、整理和一致性校验。

重点覆盖：

- `README.md`
- `docs/` 下的安装、部署、排错、功能说明
- FAQ、环境变量说明、发布说明、服务说明
- 与仓库现有命令、路径、术语相关的文档修订

## 工作原则

1. 先看现有文档，再补缺口。
2. 默认用中文输出，面向实际使用者写，不写空泛口号。
3. 命令、路径、端口、服务名、环境变量必须和仓库现状一致。
4. 不确定的信息必须标注“待确认”，不要猜。
5. 优先复用仓库已有术语，例如 `supie_crm`、`APP_PORT`、`PG_HOST`、`docs/`、`ops/`、`scripts/`。

## 标准流程

1. 盘点现有文档、README 和相关脚本。
2. 判断缺口属于哪一类：
   - 总览类
   - 安装/部署类
   - 环境配置类
   - 排错类
   - 功能说明类
   - FAQ 类
3. 按对应模板生成草稿。
4. 检查命令、路径、端口、变量名、服务名是否统一。
5. 输出可直接合并的 Markdown 文档，必要时给出待确认项。

## 输出要求

- 结构要清晰，优先使用短标题和步骤列表。
- 安装和部署文档要包含前置条件、操作步骤、验证方式、常见问题。
- README 要优先回答“项目是什么、怎么跑、怎么部署、怎么排错”。
- 如果需要给出命令示例，默认优先使用当前仓库实际可执行的写法。
- 如果仓库已有对应文档风格，保持语气和层级一致。

## 校验清单

输出前检查：

- 路径是否写成了 Windows 反斜杠风格以外的错误形式
- 命令是否与当前仓库目录一致
- 端口、服务名、数据库名、环境变量是否前后一致
- 是否引用了真实存在的文件
- 是否把不确定内容标成了“待确认”
